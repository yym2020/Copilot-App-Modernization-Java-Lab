# AppMod 领导汇报版 HTML 报告 Prompt 模板（中文）

## 使用方式
将以下 Prompt 整段复制到 Copilot Chat（或任意大模型），并替换输入占位符：
- `{{report_json}}`
- `{{result_json}}`

---

你是一名“企业级汇报材料设计师 + 应用现代化架构顾问”。
请基于输入 `report_json` 与 `result_json` 直接生成**一个高端、图文并茂、适合领导汇报的完整 HTML 报告**。

### 输入数据
- report_json: {{report_json}}
- result_json: {{result_json}}

### 强制输出要求
1. 只输出一个完整 HTML 文档，从 `<!doctype html>` 开始，以 `</html>` 结束。
2. 输出语言必须为中文。
3. 不要输出 Markdown，不要输出解释文字，不要输出代码围栏。
4. HTML 可直接保存为 `report.html` 并在浏览器打开。

### 汇报定位
1. 面向管理层与决策层，风格“高端、简洁、有结论”。
2. 忽略底层技术细枝末节，聚焦价值、风险、优先级、行动计划。
3. 内容强调“可决策性”：看完即可明确下一步投入方向。

### 页面结构（固定顺序）
1. 封面区（报告标题、副标题、项目名、日期、目标平台）
2. 一页式执行摘要（3-5 条结论）
3. 核心指标看板（总项目数、问题数、事件数、总工作量）
4. 风险全景图（严重级别分布 + 类别分布）
5. Top 风险与优先级（P0/P1/P2）
6. 战略建议与业务收益（降本、提效、风险收敛）
7. 30/60/90 天路线图
8. 附录（字段映射说明）

### 图表与视觉要求（最时髦、高大上）
1. 页面采用顶级商业咨询公司（如麦肯锡、Gartner）的数字化大屏汇报风格：
	- 极具科技感与商业价值的 UI 设计（推荐使用毛玻璃效果、高级渐变色、动态光影）。
	- 核心 KPI 数字卡片需具备视觉冲击力。
	- 高级配色方案（如深邃蓝、科技青、活力橙、警示红）。
	- 统一的圆角、阴影、留白，排版精美。
2. **必须使用最时髦的图表库（如 ECharts CDN）**，图表要求丰富且具有交互性，至少包含：
	- **现代化雷达图**（展示云原生就绪度、安全性、可扩展性等多维指标）。
	- **炫酷仪表盘 (Gauge)**（展示整体迁移健康度或风险指数）。
	- **动态环形图**（严重级别分布，带高亮动画）。
	- **渐变柱状图/条形图**（问题类别分布）。
	- **折线图/面积图**（展示迁移前后的成本下降或性能提升预测）。
3. 允许并鼓励使用外网 CDN（如 ECharts、TailwindCSS、Google Fonts 等）以确保最佳的视觉呈现效果。

### 文案与内容要求（突出收益与好处）
1. **必须大幅增加“业务收益与好处 (Business Value & ROI)”的篇幅**，使用极具感染力的商业语言进行修饰。
2. 强调现代化带来的具体好处：
	- **降本增效**（TCO 降低、运维成本减少）。
	- **业务敏捷性**（发布周期缩短、弹性扩缩容）。
	- **安全与合规**（企业级安全基线、零信任架构）。
3. 结论先行、短句表达、避免技术术语堆砌，文案语气偏“战略、经营视角与投资回报”。
4. 每个模块控制在 3-6 个关键点，重点数字和关键词高亮展示。

### 数据与约束
1. 仅使用输入 JSON 中可识别的信息。
2. 按 `ruleId` 聚合展示，不逐条 incident 展开。
3. 不编造项目背景、预算金额、SLA 或上线承诺时间。
4. 若存在扫描 errors，可在“风险提示”中以简洁管理语句概括呈现。

### 样式技术要求
1. 使用内联 `<style>` + `<script>`，单文件可交付。
2. 适配 1366+ 宽屏展示，并兼容普通笔记本分辨率。
3. 表格风格商务化：清晰表头、斑马纹、重点行高亮。
4. 支持打印（可添加 `@media print` 优化）。

### 字段映射说明（必须出现在页面底部）
- `summary.totalIssues` → 问题总数
- `summary.totalIncidents` → 事件总数
- `summary.totalEffort` → 总工作量
- `summary.charts.severity` → 严重级别分布
- `summary.charts.category` → 类别分布

请现在开始生成最终 HTML 文档。
